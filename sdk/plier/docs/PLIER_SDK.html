PLIER SDK
This document will outline the requirements and functional characteristics of the Probe Logarithmic Intensity Error Algorithm (PLIER) application programming interface (API). The PLIER algorithm is an expression analysis algorithm that is accessible from the SDK. This document describes the interfaces that are required to execute the algorithm.
GENERAL OVERVIEW
The PLIER SDK code shall allow users to execute the PLIER algorithm on any workstation that has an ANSI C/C++ compiler. The PLIER SDK executes the PLIER algorithm and provides results from the PLIER execution. It allows a user to set and or get the various parameters that are required for the execution of PLIER. The user is assumed to have a basic knowledge of C/C++ and object oriented programming. The SDK does not allocate any memory for the data inputs or outputs. It is the user's responsible to manage the memory allocation. A reference count shall be maintained in the PLIER interface object and the object shall delete itself when the reference count goes down to zero.
The PLIER SDK works as follows:
a) An initial estimate is made by the Simplified Expression Analysis (SEA) method. The various parameters labeled SEA control initialize this estimation procedure.
b) This estimate is refined by a descent method using the PLIER "goodness of fit" as the optimization criterion. This portion of the code performs the actual "PLIER" estimation of values for affinity and concentration.
c) Whenever part (b) converges, a grid search is performed to find if the values are merely a local minimum of the "goodness of fit". If a better fit is found, we return to part (b), otherwise, we exit with the current values.

The flag labeled "optimization" is not a TRUE/FALSE flag. It indicates full optimization of the Maximum Likelihood Estimator (FULL_MLE) for the value 0 (i.e. do parts (b) and (c)), and SEA for the value 1 (do not do parts b and c, i.e. do not execute PLIER!). Thus, setting the optimization flag to 0 runs quickly, but the outputs are simply the initial estimates and not the PLIER estimates.

set_sea_attenuation
Description
 Parameter Type
 Input Parameters
 Return Value
The ATTENUATION parameter controls the use of background values in SEA. ATTENUATION = 1 means that background is ignored, and PM only values are used for the initial approximation. ATTENUATION = 0 means that the value is max(0, PM-MM). Values in between "attenuate" the background smoothly between those two extremes. This is a first-order approximation to the RMA technique of taking "expected background". A value of 0.005 is a good default, leaving about 5% of the PM intensity when MM is larger than PM.
Default value: 0.005
Valid Range: 0<x<1.0
Useful Range: 10^-6<x<1.0

set_plier_probepenalty
Description
 Parameter Type
 Input Parameters
 Return Value
The PROBEPENALTY parameter is a "Bayes penalty" on affinity values being very different across probes. Larger PROBEPENALTY values indicate that probes are expected to have very similar affinities within a probeset (all near 1.0). Small values indicate that affinities should follow the observed data.
Default Value: 0.001
Valid Range: 0<x<infinity
Useful Range: 10^-6 <x < 1.0

The CONCPENALTY parameter is a "Bayes penalty" on concentration values being very different across experiments. Typically, most genes are expected not to change. Large values of CONCPENALTY (not recommended) indicate that the concentrations are expected to be similar for most genes across all experiments.
