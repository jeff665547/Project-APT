MAS5 STAT SDK - RELEASE 2

This document will outline the high level public interfaces of the MAS5 Statistical (STAT) Algorithm Application Programming Interfaces (APIs). For detailed information on the MAS5 algorithm, please refer to the Statistical Algorithms Reference Guide available on the Affymetrix web site.

RELEASE NOTES

There are some differences with the implementation in this version of the MAS5 algorithm as compared to that in the MAS5 and GCOS software. The differences include:

  • This version does not implement probe masking.
  • For comparison analyses the baseline CEL file is input instead of a baseline CHP file.
  • Differences in the precision of the results may be observed due to differences in compilers and optimization settings.
  • MAS5 and GCOS software show signal results in its display with one significant digit and p-value with 6 significant digits.

Release 1 of the SDK provided the ability to generate detection, signal, p-values for an absolute analysis and change calls, change p-values and signal log ratios for comparison analysis of expression arrays. This version was developed and tested on a Windows XP system using MSVC++.NET 2003 and RedHat Linux 8 using G++ version 3.0.

Release 2 of the SDK provided the following changes:

  • Tested on using MSVC++.NET 2003 on Windows XP and G++ 3.3 on Apple Macintosh G5 System X and Cygwin (under Windows XP).
  • Use of the File Parser SDK release 2 using memory mapping techniques for CEL and CDF file access are used.
  • Added calculations for RawQ, background and noise average, standard deviation, minimum and maximum values and corner and central + and - average intensities.

PARAMETERS

The default parameters set in the source code are from the U133A array and for the GCS3000 scanner.

 CONSTANTS

There are several constants that may need to be defined to enable compilation and execution on specific operating systems. These are :

IS_BIG_ENDIAN - Define this value if the machine represents its data in big endian format.

_INCLUDE_UNISTD_HEADER_ - Define this value if the UNISTD.H header file is used by the C++ compiler on your machine.

WIN32 - Define this value if using the MSVC++.NET compiler.

_USE_MEM_MAPPING_ - This value is used to indicate if memory mapping should be used. The value is set within the source code. Memory mapping is only implemented for BAR, BPMAP, CDF and CEL file reading.

INTERFACE DEFINITIONS

The interfaces below represent the interfaces to the statistical algorithm. Only the high level interfaces for this SDK are described in these documents. File access and detailed analysis and utility classes are not described.

CExpStatAlgSettings - This class provides storage for the algorithm parameters.

AbsStatExpressionProbeSetResultType - This type provides storage for the absolute analysis results for a single probe set.

CompStatExpressionProbeSetResultType - This type provides storage for the comparison analysis results for a single probe set.

CExpressionAlgorithmImplementation - This class provides the high level interfaces to run the expression analysis algorithm.
